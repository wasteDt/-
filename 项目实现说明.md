# Node.js + Socket.IO 实时聊天室项目实现说明

## 1. 项目概述

本项目是一个基于 Node.js 和 Socket.IO 实现的多人实时聊天室。用户可以设置昵称进入聊天室、发送实时消息、修改昵称，并查看当前在线用户。系统还会广播用户加入、改名和离开的提示。

项目采用前后端分离的事件通信方式：

```text
用户操作
  -> 客户端通过 socket.emit() 发送事件
  -> 服务端校验数据并更新在线状态
  -> 服务端通过 io.emit() 广播处理结果
  -> 客户端监听事件并更新页面 DOM
```

主要技术：

- Node.js：创建 HTTP 服务并运行后端程序。
- Socket.IO：建立实时双向连接，完成事件发送和广播。
- HTML、CSS、JavaScript：实现聊天室页面和客户端交互。
- Map：在服务端内存中保存在线用户。

## 4. 创建 HTTP 和 Socket.IO 服务

**代码位置：`server.js` 第 1-48 行、第 188-190 行**

服务端先引入文件、路径、HTTP 和 Socket.IO 模块：

```js
const path = require('path');
const fs = require('fs');
const http = require('http');
const { Server } = require('socket.io');
```

`http.createServer()` 根据请求路径读取 `public` 目录中的静态文件，并根据文件扩展名设置响应类型。第 30-34 行还检查了最终文件路径，防止客户端访问 `public` 目录之外的文件。

HTTP 服务创建后，将其交给 Socket.IO：

```js
const io = new Server(server);
```

最后通过 `server.listen()` 监听端口。默认端口是 3000，也可以使用环境变量 `PORT` 修改。

## 5. 建立客户端实时连接

**客户端代码位置：`public/client.js` 第 1 行、第 96-111 行**  
**服务端代码位置：`server.js` 第 89-90 行**

客户端调用 `io()` 与当前服务器建立连接：

```js
const socket = io();
```

服务端通过 `connection` 事件接收每一个客户端连接：

```js
io.on('connection', (socket) => {
  socket.emit('users:update', publicUsers());
});
```

其中，`socket.emit()` 只向当前新连接的客户端发送在线用户列表。客户端还监听 `connect`、`disconnect` 和 `connect_error`，根据连接状态更新页面提示。

## 6. 在线用户数据管理

**代码位置：`server.js` 第 50-79 行**

服务端使用 `Map` 保存在线用户：

```js
const users = new Map();
```

`Map` 的键是 Socket.IO 自动生成的 `socket.id`，值是用户对象。每个浏览器连接都有独立的 Socket ID，因此可以准确识别用户所属的连接。

辅助函数的作用如下：

- `cleanText()`，第 52-57 行：合并多余空格、去除首尾空格并限制长度。
- `isNameTaken()`，第 59-69 行：忽略大小写检查昵称是否重复。
- `publicUsers()`，第 71-75 行：按加入时间排序，并只返回允许客户端看到的用户字段。
- `emitUsers()`，第 77-79 行：向所有客户端广播最新在线用户列表。

## 7. 用户加入聊天室

**客户端代码位置：`public/client.js` 第 113-131 行**  
**服务端代码位置：`server.js` 第 92-114 行**

用户提交昵称表单后，客户端根据 `currentUser` 判断本次操作是加入还是改名：

```js
const eventName = currentUser ? 'user:rename' : 'user:join';
const result = await sendWithAck(eventName, name);
```

第一次提交时发送 `user:join` 事件。服务端依次完成以下处理：

1. 调用 `cleanText()` 清理昵称并限制为 20 个字符。
2. 判断昵称是否为空。
3. 调用 `isNameTaken()` 检查昵称是否已被占用。
4. 使用 `users.set()` 保存用户 ID、昵称和加入时间。
5. 通过回调向当前客户端返回操作结果。
6. 调用 `emitUsers()` 广播最新用户列表。
7. 调用 `systemMessage()` 广播用户加入提示。

服务端返回成功结果后，客户端保存当前用户，并启用消息输入框：

```js
currentUser = result.user;
setChatEnabled(true);
```

## 8. 应答回调机制

**代码位置：`public/client.js` 第 90-94 行**

项目将 Socket.IO 的 acknowledgment 回调封装为 Promise：

```js
function sendWithAck(eventName, payload) {
  return new Promise((resolve) => {
    socket.emit(eventName, payload, resolve);
  });
}
```

客户端发送数据时同时传入 `resolve`。服务端处理完成后调用 `reply()`，客户端就可以通过 `await` 得到 `{ ok, error, user }` 等结果。这种方式可以明确判断加入、改名或消息发送是否成功。

## 9. 在线用户列表更新

**服务端代码位置：`server.js` 第 71-79 行**  
**客户端代码位置：`public/client.js` 第 74-88 行、第 109 行**

当用户加入、改名或离开时，服务端调用：

```js
io.emit('users:update', publicUsers());
```

`io.emit()` 会把事件发送给所有已连接客户端。客户端监听事件并调用 `renderUsers()`：

```js
socket.on('users:update', renderUsers);
```

`renderUsers()` 更新在线人数、清空旧列表、重新创建用户列表项，并为当前用户添加 `me` 样式类。

## 10. 发送与广播聊天消息

**客户端代码位置：`public/client.js` 第 133-151 行**  
**服务端代码位置：`server.js` 第 149-173 行**

客户端提交消息表单后，先去除首尾空格，再发送 `chat:message` 事件：

```js
const result = await sendWithAck('chat:message', text);
```

服务端收到消息后执行以下逻辑：

1. 根据 `socket.id` 查询发送者。
2. 清理消息文本并限制为 500 个字符。
3. 检查用户是否已经进入聊天室。
4. 检查消息是否为空。
5. 生成包含消息 ID、用户 ID、昵称、正文和发送时间的消息对象。
6. 通过 `io.emit('chat:message', message)` 广播给所有客户端。
7. 通过 `reply({ ok: true })` 告知发送者处理成功。

消息对象定义在 `server.js` 第 163-169 行：

```js
const message = {
  id: `${Date.now()}-${socket.id}`,
  userId: socket.id,
  username: user.name,
  text,
  createdAt: new Date().toISOString()
};
```

## 11. 客户端消息渲染

**代码位置：`public/client.js` 第 37-72 行、第 110-111 行**

客户端监听系统消息和普通聊天消息：

```js
socket.on('chat:system', addSystemMessage);
socket.on('chat:message', addChatMessage);
```

`addChatMessage()` 比较消息中的 `userId` 和当前用户 ID：

```js
const isMe = currentUser && message.userId === currentUser.id;
```

如果消息是当前用户发送的，就为消息元素添加 `me` 类，通过 CSS 显示不同样式。消息正文使用 `textContent` 插入页面，不会被浏览器当作 HTML 执行，可以降低脚本注入风险。

`formatTime()` 将服务端传来的 ISO 时间转换为本地小时和分钟，`scrollToLatest()` 则让消息区域自动滚动到底部。

## 12. 修改昵称

**客户端代码位置：`public/client.js` 第 113-131 行**  
**服务端代码位置：`server.js` 第 116-147 行**

当 `currentUser` 已经存在时，客户端提交昵称表单会发送 `user:rename` 事件。

服务端先确认用户已经加入聊天室，再检查新昵称是否为空或重复。如果新昵称与原昵称相同，则直接返回成功；否则更新 `Map` 中的用户数据：

```js
const oldName = currentUser.name;
currentUser.name = name;
users.set(socket.id, currentUser);
```

更新后，服务端广播最新用户列表和改名系统提示。

## 13. 系统消息

**服务端代码位置：`server.js` 第 81-87 行**  
**客户端代码位置：`public/client.js` 第 48-54 行、第 110 行**

`systemMessage()` 统一生成系统消息，并通过 `chat:system` 事件广播。系统消息主要用于提示用户加入、改名和离开。

客户端收到系统消息后，`addSystemMessage()` 创建带有 `system` 类的元素，并将提示内容添加到消息区域。

## 14. 用户断开连接

**代码位置：`server.js` 第 175-185 行**

当用户关闭页面、刷新页面或者网络断开时，Socket.IO 会触发 `disconnect` 事件。

服务端通过 `socket.id` 找到对应用户，然后执行：

```js
users.delete(socket.id);
emitUsers();
systemMessage(`${user.name} 离开了聊天`);
```

这样可以删除离线用户、刷新所有客户端的在线列表，并广播离开提示。

## 17. 数据存储方式与项目局限

在线用户保存在 `server.js` 第 50 行定义的内存 `Map` 中，聊天消息只进行实时广播，没有写入数据库。因此：

- 服务端重启后，在线用户状态会被清空。
- 新加入的用户无法查看之前的聊天记录。
- 当前所有用户都在同一个聊天室中。
- 项目没有账号登录和身份认证功能。

后续可以增加数据库持久化、账号认证、多个聊天室、历史消息查询、私聊、文件发送和敏感内容过滤等功能。

## 18. 答辩总结

本项目的核心是基于 Socket.IO 的事件驱动通信。客户端通过 `socket.emit()` 发送加入、改名和聊天事件；服务端根据 Socket ID 识别用户，完成数据清理、身份校验和状态更新；最后通过 `io.emit()` 将用户列表、聊天消息和系统提示实时广播给所有客户端。

项目完整展示了实时 Web 应用中的连接管理、事件通信、服务端状态管理、消息广播、DOM 更新和基础测试流程。
